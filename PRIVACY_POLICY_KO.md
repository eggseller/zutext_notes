# Zutext 개인정보 처리방침

**최종 업데이트**: 2026년 1월 29일
**시행일**: 2026년 1월 29일

---

## 1. 소개

Zutext에 오신 것을 환영합니다! 본 개인정보 처리방침은 노트 작성 애플리케이션("앱")을 사용할 때 귀하의 정보를 수집, 사용, 저장 및 보호하는 방법을 설명합니다. 귀하의 프라이버시는 저희에게 중요하며, 이를 보호하기 위해 최선을 다하고 있습니다.

**핵심 원칙**:
- **로컬 우선**: 모든 노트는 귀하의 기기에 로컬로 저장됩니다
- **최소한의 데이터 수집**: 앱 기능에 필요한 최소한의 정보만 수집합니다
- **사용자 제어**: 데이터 백업 시기와 방법을 귀하가 결정합니다
- **투명성**: 본 정책은 귀하의 데이터로 무엇을 하는지(하지 않는지) 명확하게 설명합니다

Zutext를 사용함으로써 귀하는 본 개인정보 처리방침의 약관에 동의하는 것으로 간주됩니다.

## 2. 수집하는 데이터

### 2.1 귀하가 제공하는 데이터

**Google 로그인 없이 앱을 사용하는 경우**:
- **노트 콘텐츠**: 귀하가 생성한 노트, 제목, 태그, 폴더, 리마인더
- **앱 설정**: 테마 설정, 글꼴 크기, 언어 설정
- **저장소**: 이 모든 데이터는 **귀하의 기기에 로컬로** SQLite 데이터베이스에 저장됩니다

**Google 로그인을 사용하는 경우 (선택사항)**:
- **Google 계정 정보**:
  - 이메일 주소
  - 표시 이름
  - 프로필 사진 URL
  - Google OAuth ID
- **목적**: Google Drive 백업 기능을 위한 인증 전용

### 2.2 자동으로 수집되는 데이터

**충돌 및 성능 데이터** (Firebase Crashlytics 통해):
- 기기 모델 및 제조사
- 운영 체제 버전
- 앱 버전
- 충돌 로그 및 스택 추적
- **개인 식별 정보는 포함되지 않습니다**

**사용 분석** (Firebase Analytics 통해):
- 앱 실행 및 세션 지속 시간
- 화면 조회 (방문한 화면)
- 기능 사용 (예: 생성된 노트, 추가된 태그)
- **노트 내용은 절대 수집되지 않습니다**

### 2.3 수집하지 않는 데이터

저희는 다음을 수집, 저장 또는 전송하지 **않습니다**:
- ❌ 귀하의 노트 내용을 저희 서버로 (귀하의 Google Drive에 백업하지 않는 한)
- ❌ 위치 데이터
- ❌ 연락처 또는 전화번호
- ❌ 사진 또는 미디어 (Google 프로필 사진 제외)
- ❌ 브라우징 기록
- ❌ 금융 정보
- ❌ 건강 데이터

## 3. 데이터 사용 방법

### 3.1 노트 콘텐츠 (로컬 저장소)

- **저장 위치**: 귀하의 기기에만 암호화된 SQLite 데이터베이스에 저장
- **사용 목적**: 노트 표시, 편집, 정리 및 검색
- **서버 전송**: Google Drive에 백업하기로 선택하지 않는 한 절대 전송되지 않음

### 3.2 Google 계정 정보

- **사용 목적**: Google Drive 백업을 위한 인증 및 권한 부여
- **저장**: 이메일, 이름, OAuth ID만 로컬 데이터베이스에 저장
- **공유 안 함**: 제3자와 공유되지 않으며 마케팅에 사용되지 않음

### 3.3 충돌 및 분석 데이터

- **사용 목적**:
  - 버그 식별 및 수정
  - 앱 성능 개선
  - 기능 사용 파악으로 개선 우선순위 결정
- **익명화**: 모든 데이터는 집계되고 익명화됨
- **보유 기간**: 60일 후 자동 삭제 (Firebase 기본값)

## 4. 데이터 저장 및 보안

### 4.1 로컬 데이터 저장

- **위치**: 모든 노트는 기기의 로컬 SQLite 데이터베이스에 저장
- **접근**: 귀하와 앱만 접근 가능
- **보호**: 앱은 기기 잠금 해제 (지문, PIN 등) 필요

### 4.2 백업 암호화 (AES-256-GCM)

Google Drive에 백업하기로 선택한 경우:
- **암호화**: 모든 백업 파일은 **AES-256-GCM** (군사급) 암호화
- **암호화 키**: 사용자별 고유, Google OAuth ID + 비밀 솔트에서 파생
- **보안**: 암호화된 파일은 귀하의 Google Drive에 저장 (저희 서버 아님)
- **접근**: 귀하만 백업을 복호화 가능

### 4.3 데이터 전송

- **HTTPS**: Google 서비스와의 모든 통신은 HTTPS 사용
- **제3자 서버 없음**: 노트 저장을 위한 서버를 운영하지 않음
- **직접 연결**: 백업은 Google Drive API에 직접 연결

## 4.4 데이터 삭제 안내
저희는 사용자의 데이터를 자체 서버에 저장하지 않으므로, 사용자의 정보를 삭제할 수 있는 "데이터베이스"가 없습니다. 사용자는 자신의 데이터를 완전히 관리할 수 있습니다.

- **로컬 데이터 삭제**: 앱을 제거하거나 Android 시스템 설정에서 "데이터 삭제" 옵션을 사용하세요.
- **백업 삭제**: 앱 설정에서 "백업 삭제" 기능을 사용하여 Google Drive에서 파일을 삭제하세요.
- **앱 접근 권한 취소**: [Google 계정 보안 설정](https://myaccount.google.com/permissions)에서 언제든지 앱의 Google 계정 접근 권한을 취소할 수 있습니다.
- **수동 삭제**: Google Drive 저장소에서 앱의 데이터 폴더를 수동으로 직접 삭제할 수 있습니다.
- **지원**: 데이터 관리 또는 삭제에 대한 질문이 있는 경우 개발자에게 **<sunjjoo@gmail.com>**으로 문의하세요.

## 5. 제3자 서비스

저희는 다음 제3자 서비스를 사용합니다:

### 5.1 Firebase (Google 제공)

**사용 서비스**:
- **Firebase Crashlytics**: 충돌 보고 및 진단
- **Firebase Analytics**: 앱 사용 분석
- **Firebase Authentication**: Google 로그인 통합

**공유 데이터**: 기기 정보, 충돌 로그, 익명화된 사용 데이터
**개인정보 처리방침**: https://firebase.google.com/support/privacy
**데이터 처리**: Google의 개인정보 보호 정책에 따라 관리

### 5.2 Google Drive API

**목적**: 선택적 클라우드 백업 저장소
**저장 데이터**: 귀하의 Google Drive에 암호화된 데이터베이스 백업 파일
**접근**: 귀하만 Drive 파일에 접근 가능
**개인정보 처리방침**: https://policies.google.com/privacy

### 5.3 Google 로그인

**목적**: Google Drive 백업을 위한 인증
**수신 데이터**: 이메일, 이름, 프로필 사진, OAuth ID
**개인정보 처리방침**: https://policies.google.com/privacy

**중요**: 저희는 마케팅 목적으로 귀하의 데이터를 판매, 임대 또는 제3자와 공유하지 않습니다.

## 6. 귀하의 권리

### 6.1 데이터 접근

- **로컬 노트**: 앱에서 언제든지 모든 노트에 접근 가능
- **백업 파일**: Google Drive에서 접근 가능 (백업을 활성화한 경우)

### 6.2 데이터 편집 또는 삭제

- **노트 편집**: 앱 내에서 언제든지 가능
- **노트 삭제**: 개별 노트 또는 전체 폴더 삭제 가능
- **백업 삭제**: Google Drive에서 직접 백업 파일 삭제

### 6.3 데이터 내보내기

- **노트 공유**: 개별 노트를 텍스트로 내보내기
- **백업 파일**: Google Drive에서 암호화된 .db 파일 다운로드
- **향후 기능**: CSV/PDF로 내보내기 (v2.0에서 계획)

### 6.4 분석 거부

- **방법**: 설정 → 개인정보 → 분석 비활성화 (출시 예정)
- **현재**: v1.0에서는 분석을 비활성화할 수 없음 (단, 데이터는 익명화됨)

### 6.5 계정 삭제

- **방법**: 앱을 제거하고 Google Drive에서 백업 파일 삭제
- **효과**: 제거 시 모든 로컬 데이터가 영구적으로 삭제됨
- **계정 없음**: 사용자 계정을 유지하지 않으므로 서버 측 삭제 불필요

## 8. 국제 사용자 및 GDPR

### 8.1 데이터 위치

- **로컬 저장소**: 귀하의 기기에 저장 (귀하가 있는 곳 어디든)
- **백업**: Google Drive에 저장 (Google 서버, 지역에 따라 다름)
- **Firebase**: Google이 데이터 처리 (국제 전송 포함 가능)

### 8.2 GDPR 권리 (EU 사용자)

일반 데이터 보호 규정(GDPR)에 따라 EU 사용자는 다음 권리를 갖습니다:
- ✅ **접근권**: 데이터 사본 요청 (앱의 내보내기 기능 사용)
- ✅ **정정권**: 부정확한 데이터 수정 (앱에서 노트 편집)
- ✅ **삭제권**: 데이터 삭제 (앱 제거, Drive 백업 삭제)
- ✅ **이동권**: 데이터 내보내기 (백업 기능)
- ✅ **거부권**: 분석 거부 (설정에서 비활성화 - 출시 예정)

이러한 권리를 행사하려면 아래 이메일로 연락하세요.

### 8.3 처리의 법적 근거 (GDPR)

- **로컬 데이터**: 정당한 이익 (앱 기능 제공)
- **Google 로그인**: 사용자 동의 (로그인 선택)
- **Firebase Analytics**: 정당한 이익 (앱 품질 개선)
